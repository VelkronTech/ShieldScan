# Security Policy

## Supported Versions

Only the latest minor version is supported for security updates.

## Reporting a Vulnerability

Please do not open public issues for security vulnerabilities.

1. Create a private report by emailing your security contact address.
2. Include reproduction details, affected files, and impact assessment.
3. Provide proof-of-concept input when possible.

## Response Targets

- Initial acknowledgment: within 72 hours.
- Triage decision: within 7 business days.
- Patch timeline: based on severity and exploitability.

## Disclosure

Coordinated disclosure is preferred. Public disclosure should occur after a fix is available or a mutually agreed date.
